疯狂安利 工具篇

安利一款手机抓包软件

发表于 2018-11-21 • 分类于 疯狂安利 , 工具篇

前言

很久之前就想试试手机抓包

记得前年的过年期间，QQ 有个下拉刷红包活动，当时配的文案是，打出的 Combo 越高，获得奖励的几率越大。

后面无意间看到过一篇文章，博主一分钟刷了 上千的 Combo，手指都划断了，但奖励什么的依旧寥寥无几，一气之下抓个包，发现刷了很多次，就发了大概几十次的请求，这就解释了为什么中奖率很低了，TX 还是会玩的…

从那时候起就想了解一下手机抓包，但这一拖就是两年后了(。_。)

这里要感谢测试的小伙伴新姐，在新姐帮助下，很快就完成了第一次手机抓包，不得不说面对面的教学，学习效率真的高。

Charles

今天安利的这款手机抓包软件叫做 Charles，又名青花瓷。但并不是翻译过来叫做青花瓷，而是 软件的 Icon 像个青花瓷的样子，所以因此得名。

好了废话不多讲，Here We GO~

至于怎么下载、安装、激活 啥啥啥的我就不多BB了，稍微搜一下就能找到的，我们直接进入正题，开始操作

简单总结了一下，我们的准备步骤：

  1. 确保 手机 和 Charles 运行机器的地址在同一网段下(最简单的方法就是连上同一个wifi…)
  2. 配置 Charles 监听端口
  3. 配置 HTTPS 代理
  4. 配置 证书 代理
  5. 移动设备安装信任证书

第一个就不多说了，我们从第二开始~

配置 Charles 监听端口

先简单解释一下这个操作，我们要给手机抓包，其实就是在手机和访问目的地之间设置了一个第三者，我们所有发往目的地的请求 以及 从目的地回来的数据，都会经过第三者然后再给我们。它会帮我们记录下请求与响应信息，同时也能拦截、修改我们发送出去的数据，这也就是我们抓包的目的所在了。

举个简单的栗子：

假设我们想用青花瓷抓取 微信数据包，那么这里手机只需要装好 微信，并打开微信应用点点点就好了；代理就是指青花瓷；目的地就是指 微信服务器啦。

所有发送给 微信服务器的请求 以及 从那过来的响应 都会先经过代理，然后再从代理转发出去，这样代理就能记录 请求 和 响应的信息了，我们只需要在代理中操作就好了。

ok，简单了解之后就可以来配置青花瓷啦。

打开菜单栏中的 Proxy -> Proxy Settings

这样配置就表示所有经过 8888 端口的 http 请求都会被监听到

配置 HTTPS 代理

由于目前 Https 的普及，几乎所有应用都走 https，所以你如果要抓包的话，这个也是必不可少的。

上面它也说了，只有下面被列出来的地址才会进行代理。简单说就是 你可以选择部分主机的流量进行监听

我们看这里有两个参数：Host 和 Port

先说 Host 吧，举个栗子，你有两个手机都需要抓包，他们都在一个网段下拥有不同的 IP地址(1.1.1.1 和 1.1.1.2)

如果这里你设置成:

Host: 1.1.1.1
Prot: *

那么 1.1.1.2 这个手机的流量就不会被代理了。

再聊聊 Port，我猜这里应该是这样(没错，就是我猜的 （ ￣ー￣）)，依然举个栗子

比如你的手机两个应用，微信 和 网易云，它们对应两个固定的端口(2233 和 3322)，如果这里你配置为：

Host: 1.1.1.1
Port: 2233

这样就只会抓取微信(2233)的数据信息啦。

我记得应用 监听端口 和 系统发送请求出去的端口好像没什么关系，所以才会觉得这样举例有点不合适….

配置 证书 代理

证书代理 和 Https 的设置方式相似，选择 Client Certificates Tab页，照着上面的逻辑操作一下就好了。

打开菜单栏中的 Help -> SSL Proxy

我猜应该可能是这些应用拒绝了青花瓷的访问信息，所以导致无法抓到数据包。于是在网上找找资料发现有人提到过这个问题。

Android Developers Blog

这是 2016年 Google 发表的 Android 开发文档关于信任证书的问题。这里面提到这样一句话：

Apps that target API Level 24 and above no longer trust user or admin-added CAs for secure connections, by default.

针对于 API Level 在 24 及其以上的 APP 来说，在进行安全连接(https)时，不再信任 用户 和 管理员自己添加的证书。

但是我测了一下自家的 APP的抓包就没有什么问题。有时间请教下 Android 的小伙伴们。
