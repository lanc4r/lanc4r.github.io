浅拷贝

浅拷贝主要涉及到两个方面：

  • 基本类型的值拷贝
  • 引用类型的地址拷贝

目前有三种方法实现，我们来了解一下

深拷贝

理解深拷贝只要对比起浅拷贝就很容易啦：

  • 基本类型的值拷贝
  • 引用类型的值拷贝

对于基本类型，和浅拷贝一样都是值的传递，但对于引用类型，深拷贝使用的也是值传递(实际上是传递了新的地址)，其实也就是重新开辟一块内存空间存放相同的值。因为有这个重新开辟内存空间的动作，所以深拷贝比起浅拷贝需要消耗更多的时间和空间。

思考

这种方法可以实现深拷贝，但是也存在着很大的弊端，我们想象这样一种情况：

A -> B，A 中有个属性为 B
B -> C，B 中有个属性为 C
C -> D，C 中有个属性为 D

当我们想 拷贝一个 A 时，需要从 D 开始，沿着继承树分别重写 D、C、B 的 clone 方法，当这个嵌套层更深的时候，会变得很麻烦，你需要写大量的代码，那样肯定会累死的…

这种方法只适用于嵌套层次不是很深的情况，那么当深层嵌套的情况发生我们怎么做呢，看看第二种方式 (。・・)ノ
