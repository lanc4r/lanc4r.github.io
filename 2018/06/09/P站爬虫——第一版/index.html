P站爬虫 (～￣▽￣)～

好久好久好久之前就打算写的，远到我都记不清具体是多久之前了。终于在毕业答辩前夕，写出了这个半成品，不过还是很开心啦~

这里先在博客里记录一下，主要是怕答辩完浪几天后啥都忘了….

好了，废话不多说，Here We Go~

分析

这次我们的目标是 P站 的插画，主要包含的位置是：推荐、日榜、周榜、月榜。(真的太好看啦ヾ(≧▽≦*)o，总感觉自己对图有种莫名的执着…)

首先来看看 推荐

在推荐作品板块里我们点击浏览更多，然后抓包我们可以发现这样一个神秘的地址：

https://www.pixiv.net/rpc/recommender.php?type=illust&sample_illusts=auto&num_recommendations=1000&page=discovery&mode=all&tt=d9e6e15ed29ffa52ccc8a0e4f22445d1

看上图，没错，这个接口能获取当前推荐页面的 ID值集合，其实这里就和之前 写的那个 B站爬虫下载器 有相同的用法了，我们可以不断的请求这个接口，来获取更多的图片，当然效率很慢就是了，不过可以省去很多麻烦，也算一种取巧的方式啦。

再分析 日榜、周榜、月榜

日榜、周榜、月榜 获取方式都一样的，这里就用 日榜举例了~

Rank榜的图片需要分两段获取，直接访问其地址只能获取 排名前50的图片，后面需要访问新的接口来获取新的图片。

前 50 获取

直接访问其地址，将内容保存下来，我们发现，前 50 的图片信息全部都包含在里面了~

50 之外数据获取接口

在日榜页面将右侧滚动条一直往下滑动，当一直浏览到五十开外的数据的时候，抓包就会发现这个接口

https://www.pixiv.net/ranking.php?mode=daily&content=illust&p=2&format=json&tt=d9e6e15ed29ffa52ccc8a0e4f22445d1

注意到其中传递的参数，p，目前值为2，所以我们可以推断出 通过改变p值来获取当前Rank榜更多的数据

如何获取图片

可以发现，前面我们所有的操作都是取得图片的 ID值，那么具体应该怎么构造 url 来获取图片呢，这里我就懒得截图分析过程了，自己去看一下页面结构很快就能明白的~

获取图片有两种方式：单图页面 和 多图页面

单图页面

访问 https://www.pixiv.net/member_illust.php?mode=medium&illust_id=yourid，在返回的内容中找到 class = original-image 的 img 标签，其 data-src 属性就是原图地址

多图页面

访问 https://www.pixiv.net/member_illust.php?mode=manga&illust_id=yourid，在返回的内容中找到 class = image ui-scroll-view 的 img 标签(有多个)，其 data-src 属性就是原图地址

开码

分析到以上情况，接下来就是写代码时间了，这里我们用到 requests 和 BeautifulSoup 两个库来抓取和解析爬取的内容，然后使用 PIL 保存图片。

HTMLDownloader

html 页面内容下载器，用于返回 页面内容，包括二进制数据

urls.json

主要是保存 url 地址啦

HtmlParser

解析 Downloader 返回的页面内容，获取关键信息(后面只贴出关键代码)

UrlManager

管理 ID，记录 待爬取的ID 和 以爬取的ID，其实叫 IDManager 更合适(懒得改了 -_-)

ImageDownloader

图片下载器，将 HtmlDownloader 返回的 二进制数据保存为图片

Mainer

总调度器，调用上面所有的类完成任务

总结

如上，核心代码就是这样子，没有高端使用技巧，没有花里胡哨的东西，并且还很蠢！

发现了吧，使用的时候需要自己先登录上 P站，然后把 cookie 拷下来放上去…

所以目前也不能算作第一版吧，代码等我优化之后再上传到 Github 上，之后再分享出去

优化

  • 登录部分得完成，之前也写了一点，但是发现在访问完主页之后获取的 cookie 不完整，还有一段 js 代码来构造剩余部分 cookie，当时觉得麻烦，也没有太细看，后面再撸这段 js 吧。
  • 找个算法优化一下 UrlManager，主要是优化判断 ID 是否在已爬取的 ID列表中
  • 目前超时率蛮高的，后续看能不能操作一下
  • 再做一个 Chrome Extension 配合一下爬虫的使用
