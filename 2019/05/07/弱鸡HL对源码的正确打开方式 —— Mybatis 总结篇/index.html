Mybatis 总结

这里主要简单聊聊 Mybatis 执行流程 以及 两个小技巧

Mybatis 执行流程

前两篇我们分别说了 Mybatis 解析配置文件 和 执行SQL(包括结果映射) 的过程，这里大致画一下执行的流程(真的就是很简洁的画了画….)

简单BB

  1. 首先是创建一个 SqlSessionFactory 对象工厂，这个工厂主要包含两部分东西：

    • SqlSession —— 用于执行 SQL 语句(例如：selectList、selectOne、update、delete 等)
    • 配置文件对象 Configuration —— （用于保存 XML 的配置信息，还有各种各样的类型信息都放入这里，后面需要用到的
      情况就来这里取）
      • 通过 XMLConfigBuilder 解析 Mybatis 配置文件，逐个标签解析，并将解析后的数据都存放到 Configuration 对象中
      • 通过 XMLMapperBuilder 解析 SQL 映射文件，逐个标签解析，并将解析后的 SQL 语句放到 BoundSql 对象中。BoundSql 是存放在 MappedStatement 中，而 MappedStatement 是存放在 Configuration 对象中

  2. 通过 SqlSessionFactory 创建执行器 SqlSession (貌似一般都是 DefaultSqlSession)，其中主要包含两部分数据：

    • Configuration —— 配置文件，包含数据
    • Executor —— Sql 语句执行器

  3. 执行过程首先通过 Configuration 获取 MappedStatement 对象(上面介绍过，它包含 BoundSql 也即是 SQL映射文件中的数据)

  4. 通过 Executor 执行查询 (Executor 是接口，具体由实现类来完成，例如：BaseExecutor)，这里需要的 SQL语句来自步骤三的 MappedStatement 对象。

  5. 在 Executor 中创建 Connection

  6. 在 Executor 创建 StatementHandler，然后由其创建 Statement 对象 (StatementHandler 是接口，具体由实现类来完成，例如： BaseStatementHandler)

  7. 对创建的 Statement 语句进行参数化，这里通过 ParameterHandler 来操作 (ParameterHandler 是接口，具体由实现类来完成，例如： DefaultParameterHandler)

  8. 然后是查询结果的映射
    由 ResultSetHandler 来对查询结果进行映射，ResultSetHandler 是一个接口，具体操作由其实现类完成，例如：DefaultResultSetHandler。

2. 再说一个 #{} 和 ${} 的区别：

这两者的比较相信大家并不陌生，在最开始学习 Mybatis 的时候就会说，推荐使用 #{} 代替 ${}，因为 #{} 中设置的参数会变成 ? 占位符进行预处理，而 ${} 会直接替换传入的文本而不进行转义。所以使用 #{} 代替 ${} 可以在一定程度上防止 SQL 注入攻击。

但有时候是需要使用 ${} 的，比如这个情况：你可能会需要通过某个表不同的字段来查询数据，这样你就需要写多个查询语句
