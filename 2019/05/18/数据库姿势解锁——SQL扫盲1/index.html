SQL常见用法

这次先介绍两种查询的方法。

连接查询

首先聊聊这个连接查询，连接查询最核心的思想就是：一次查询需要的结果在多张表而非一张表中，这时候就需要使用到连接查询，也即是连接多张表进行查询(通常是需要横向扩充字段)。

常见分类：内连接查询、外连接查询、交叉连接查询。

内连接查询

内连接查询生成的数据是需要连接条件满足才会显示出来。也就是说只有条件满足两边的数据才会显示出来，注意没有满足连接条件的数据是不会显示的哦，这里主要是和外连接做区别的(后面会讲到)。

由上我们可以对自然连接得出以下结论：

  1. 自然连接算是一种特殊的等值连接，它与等值连接的区别在于：它会将重复的列去掉，并且自动的将两张表所有列名相同的列进行等值连接匹配。当没有相同列名的列的时候，就会将量表的结果进行笛卡尔积(左表条数 x 右表条数)。
  2. 自然连接没有连接条件(简单说就是不能用 on 关键字加上连接条件)，因为它会自动的寻找俩表列名相同的列进行连接，这时候如果再使用 on 进行条件连接就会报错了。

外连接查询

外连接与内连接的区别就是：外连接能够将不满足连接条件的那部分数据也展示出来。

举个简单的栗子
比如 A表 只有一个字段 ID，并且只有 1、2、3 三个值
B表 也只有一个字段 ID、其中含有 1、2、5 三个值

这时候如果使用内连接的话，那么只能匹配两条数据出来，即俩表ID为 1、2 的数据，但如果使用外连接就可以把 俩表ID 不相等的数据都匹配出来，这时候就有个问题了：

既然是不相等的数据能被匹配出来，现在我们有 3 和 5 两个值，那么到底是让谁显示出来呢，基于这个让谁显示出来的问题，我们把外连接分为了：

  • 左外连接：当连接条件不匹配的时候，左表数据正常显示，右表数据使用 NULL 值补充。简单说就是左表的数据全部查出来。
  • 右外连接：当连接条件不匹配的时候，右表数据正常显示，左表数据使用 NULL 值补充。简单说就是右表的数据全部查出来。
  • 全外连接：当连接条件不匹配的时候，俩表的数据都正常显示，左表数据 和 右表数据 都会在不匹配情况使用 NULL 值补充。简单说就是俩表的数据都会查询出来，只是它们不匹配的地方全部使用 NULL 值来补充。

联合查询

前面讲的连接查询都有一个特点：查询结果列数增加了(横向扩展)，毕竟是连接了两个表嘛，所以相当于把两个表每一行的数据横向拼接了起来。而我们的联合查询也是用于多表数据的组装，但是它并不是横向拼接，而是纵向拼接，既然是纵向拼接这就表示 俩表查询出的列数必须是相等的！

联合查询不像连接查询，它没有那么多花里胡哨的东西，只有两种用法 Union 和 Union All，简单记就是 Union 会去重，而 Union All 不会，看一个栗子就明白啦：
