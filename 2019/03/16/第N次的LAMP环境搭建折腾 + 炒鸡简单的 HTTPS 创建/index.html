一个炒鸡简单的 HTTPS 创建

事情依旧是源于公司的分享，我无意间挖到一个公司网站的 XSS 漏洞，想着分享的时候当做栗子演示一下。但是当我在利用漏洞访问我服务器上资源的时候，发现由于 HTTPS 的安全策略，是不会加载 HTTP 协议的资源，没办法，没有条件只有自己动手创建条件了…

准备步骤

  • 获取 SSL 证书 (申请/购买/自制)
  • 绑定域名
  • 搭建环境 (Apache/Nginx/Tomcat …)
  • 测试连接

获取 SSL 证书 (白嫖/购买/自制)

获取 SSL 证书是第一步，只有有效的证书才会被浏览器认定这是一个安全的连接，也就是我们的连接前面有一个 小锁头

这里我只讲讲如何如何白嫖一个为期一年的证书，因为我只是为了此次演示用所以就只搞了这一个，大家根据自己需求选择吧，如果是自制的话去网上搜一家教程，还是蛮多的。

这里我选择阿里云的平台进行证书的申请 SSL证书，访问之后我们需要选择 免费型 DVSSL

支付完成之后我们就需要开始证书的申请了，我们需要填入证书绑定的域名地址，这样主要看一下 域名的验证方式 和 CSR生成方式的选择，我们选择 自定DNS 验证 和 系统生成

填写完成之后就会跳转到验证的页面，下面是我自己的截图，对应绑定的域名为：

聊聊国内的服务器

我们知道国内的服务器是需要备案才能设置域名解析的，这是由于国内服务器都默认监听了 80 端口，所以当你不备案做了域名解析就会提示你需要备案，但是如果是 HTTPS 貌似默认不会被拦截到，所以这也算是绕过备案的一种姿势吧。
