连接主机

Windows 推荐 XShell、putty，我就不多bb了，网上一大堆的资料。

然后 Linux 我们直接使用 自带的 ssh 连接就好啦，在命令行中输入命令：

ssh -p 端口号 服务器用户名@主机ip

比如: ssh -p 22 root@101.202.101.202，正常情况需要你信任一下当前的主机，输入 yes 就好，接着就是输入密码了，去 Vultr 的管理后台就能找到。

在终端输入: vim /Users/lanc4r/.ssh/known_hosts

在其中找到你的 认证 Key ，删掉它，然后重新使用 SSH 连接

  • 如果提示 WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!，解决步骤跟上面的方式一样，其实它都已经告诉你这个解决办法了…

Tips

先简单说一下 SS 的两个常见模式的区别：

  • PAC代理模式(又叫自动模式)
  • 全局代理模式

先说说全局模式吧，全局模式表示所有的地址都走代理，比如我们在墙内访问什么 百度、新浪，这时候依然走的代理，也就是说我们 访问墙内的东西会先发请求去墙外，然后由墙外向墙内发起请求，如果没有特殊需求不建议使用全局模式…

然后是 PAC 代理模式，它其实存储了一份地址名单，如果你请求的地址存在于名单上就会走代理，如果不存在于名单上就不会走代理，这样就会避免绕出墙来访问墙内世界的这种操作。

这个代理规则是有两份名单来决定的，其中线上有一份公用的，几乎能满足你 90% 的代理需求，当然那 10% 也是支持自定义的，点击 SS 选择 编辑PAC用户自定规则，然后根据不同的操作系统自己设置就好了，具体的规则自己去了解吧。

说了这么多要说一下我们的 Tips，如果你开着代理发现某个网站访问不了，这时候不要急，先切换至 全局模式 试一下，如果全局模式能点开，这时候就需要我们上面讲的自定义用户 PAC 的方法啦。

当然如果切换至全局模式也访问不了的话，这时候可以选择 ping 一下这个地址，看是不是它的服务器挂了…
