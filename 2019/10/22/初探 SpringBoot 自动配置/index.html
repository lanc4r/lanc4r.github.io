初探 SpringBoot 自动配置

在接触 SpringBoot 之前用得最多的就是 SSM 框架，每次要搭一个简单的 Web 项目时候都需要导入一大堆东西、配置一大堆的东西，web.xml 的 DispatcherServler，视图解析器配置，以及 各种书写配置文件，总的来说，有点麻烦…

但 SpringBoot 可以不用写配置文件，不过并不是指 SpringBoot 不需要配置， 因为SpringBoot 是依赖于 Spring 的，所以，必然框架在某个地方存在 Spring 的配置的，只是对于来说隐藏了。接下来要做的就是看看 SpringBoot 到底是怎么完成的~

SpringBoot

spring-boot-starter

首先简单介绍一下 SpringBoot 一个很关键的东西：场景启动器。它最大的作用就是帮助我们导入运行是需要的组件，以 spring-boot-starter-web 为例，我们 pom 文件长这样：

这里的 spring-boot-starter-web 其实就是一个 场景启动器啦，它的主要作用就是帮助我们导入 web 模块运行需要的组件，我们可以点进 spring-boot-starter-web 的 pom 文件看一下

发现其中导入了运行必须的组件 比如：spring-web、spring-webmvc 以及 Tomcat 等等。

SpringBoot 将所有的功能场景都抽取出来，做成了一个个的 starter(启动器)，我们只需要在项目中引入对应的 starter，相关场景所有的依赖都会自动导入进来。要用到什么功能就导入对应的启动器就好了。

再多说一句，starter 控制导入哪些组件，那么这些组件的版本呢？所需的版本就放在 spring-boot-starter-parent 的 spring-boot-dependencies 中

Spring主程序类

@SpringBootApplication SpringBoot 应用注解标注在某个类上就表示这个类是 SpringBoot 的主配置类，SpringBoot 就应该运行这个类的 main 方法来启动 SpringBoot 应用。

我们知道在运行 Spring 项目的时候，我们需要做很多的操作，比如 配置包扫描路径，将对应注解的类加入到 Spring 容器中；配置视图解析器 等等，这些操作肯定是在 Spring 项目启动过程中完成的，SpringBoot 也不例外，肯定免不了这些操作，接下来我们将他们找出来，首先看看这个 @SpringBootApplication 注解

@SpringBootConfiguration

其中有一个 @SpringBootConfiguration 注解，SpringBoot 配置注解，了解一下

其中有一个很熟悉的注解 @Configuration，这个注解是 Spring 的注解，被此注解修饰的类就表示一个配置类(可以理解为一个配置文件，只是把配置文件的数据放到了类中存储)，其内部使用了 @Component 注解进行修饰，这就是表示它在 Spring 启动的时候会被 Spring 扫描到放入 Spring 容器中，受 Spring 的管理。

@EnableAutoConfiguration

第二个注解是 @EnableAutoConfiguration，开启自动配置，嗯，听名字跟我们需要的东西有关，自动配置，可以理解为不用手动写配置的意思啦。当我们使用这个注解的时候，SpringBoot 就自动帮我们设置好运行需要的配置信息。

这里有两个很关键的注解：@AutoConfigurationPackage 和 @Import(EnableAutoConfigurationImportSelector.class)

先看看 @AutoConfigurationPackage

自动配置包

这里又使用了一个 @Import(AutoConfigurationPackages.Registrar.class) 注解修饰，@Import 是 Spring 的一个注解，它的作用是向 Spring 容器中导入一个组件，于是关键就在于 AutoConfigurationPackages.Registrar.class 这个类。

打个断点在 registerBeanDefinitions() 方法上，查看获取到的 packageName 是什么

发现得到的包名是 "com.lanc4r" 其实也就是由 @SpringBootApplication 注解修饰的类所在路径啦。至此可以得出结论 @AutoConfigurationPackage 的作用就是将 @SpringBootApplication 注解修饰的类(主配置类) 所在包及子包 路径下所有的组件扫描到 Spring 容器中。
