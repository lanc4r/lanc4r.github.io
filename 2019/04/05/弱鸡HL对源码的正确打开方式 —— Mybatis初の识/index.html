仔细看看我们的代码，我们做的事情很简单：

  • 定义了一个 读取 xxx配置文件的读取流 inputStream
  • 获取 SqlSession，传入 SQL映射文件的 ID，通过其来找到我们自己的 SQL语句，解析后执行
  • 将结果存储在我们自定义好的 News 类中，并返回

关键点就在这个 SqlsessionFactory 和 SqlSession 了，它们至少完成了以下三个操作：

  • 解析了配置文件 mybatis-config.xml，获取连接数据库的相关信息，并创建了数据库连接
  • 解析了我们自定义的 SQL 映射文件 NewsMapper.xml，获取并解析我们自定义的 SQL
  • 执行 SQL 语句，将结果存入到我们自定义好的类 News 中，返回其值

在初识篇的内容中我们将通过 Debug 的方式来看看 以下两个操作发生在 Mybatis 的什么地方

  • 解析配置文件 mybatis-config.xml，获取数据库的连接信息
  • 解析我们自定义的 SQL 配置文件，并解析生成 SQL 语句
